This is a demo of "Cosmic Bureaucracy: Lost in Translation", where your task is to register at an Immigration Office in a foreign country.

The intention behind the game was to explore certain aspects of applying large language models to procedural generation and generative storytelling:

  • How well can they impersonate a specific job role (immigration officer in this case)?
  • How well can an LLM work with constructed languages?
  • How well can I restrain the model to adhere to a predefined set of game rules?

Important notes:

  • Running this game costs me money; about $0.5 to $2 per game session. The game will be shutdown eventually, right now I'm not planning to continue development.
  • The game uses OpenAI API under the hood, do not submit anything that you do not want to share with them.
  • For the same reason, I do not control the output of the model perfectly, it can give unwanted responses, be mindful of that.
  • The game has bugs, and the officer's logic is not perfect. The API is not very stable. There are many flaws. That's the reality of it (I had to cut down certain things to fit into the time I have and avoid scope creep).

Initially the game was intended to be run on what's now a hard difficulty, but at the last minute, I've decided to add lower difficulties. They can be changed on the fly, future responses will take the new settings into account

  • Hard -- nothing is translated for the frontend, you need to decipher the language yourself
  • Normal -- some things are occasionally translated into English, but most phrases are kept untranslated
  • Easy -- all the officer's phrases are translated into English
  • Debug -- everything has an English translation, including an internal translation of what you submit
